# 服务间业务流程

## 1. 服务间通信总览

```
┌──────────┐     HTTP/REST      ┌──────────────┐
│ manager- │◄───(Bearer token)──│  manager-web  │
│ api      │◄───(Bearer token)──│ manager-mobile│
│ (Java    │                    └──────────────┘
│  8002)   │     HTTP/REST
│          │◄───(Bearer secret)─┐
│          │                    │
│          │    WebSocket       │
│          │───(JWT+secret)──►┌─┴────────────┐
│          │                  │ xiaozhi-      │
│          │◄──HTTP/REST──────│ server        │
│          │  (Bearer secret) │ (Python 8000) │
└────┬─────┘                  └───┬───────────┘
     │                            │
     │ HTTP(anon)                 │ WebSocket(JWT)
     │                            │
┌────▼─────┐              ┌──────▼──────────┐
│ ESP32    │──HTTP(OTA)──►│ OTA端点         │
│ 设备     │──WebSocket──►│ (Java或Python)  │
└──────────┘              └─────────────────┘
```

## 2. 通信矩阵

| 序号  | 源                      | 目标                     | 协议                 | 认证方式                 | 用途                  |
| --- | ---------------------- | ---------------------- | ------------------ | -------------------- | ------------------- |
| 1   | ESP32设备                | manager-api/Python     | HTTP POST          | 无(anon)              | OTA上报               |
| 2   | ESP32设备                | Python(xiaozhi-server) | WebSocket          | Bearer JWT(HMAC)     | 语音对话                |
| 3   | Python(xiaozhi-server) | manager-api            | HTTP POST          | Bearer server.secret | 拉取配置/上报记录           |
| 4   | manager-api            | Python(xiaozhi-server) | WebSocket          | JWT + content.secret | 管理指令(update_config) |
| 5   | manager-api            | RAGFlow                | HTTP(S)            | Bearer api_key       | 知识库操作               |
| 6   | manager-api            | 阿里云短信                  | HTTPS SDK          | AK/SK                | 发送短信                |
| 7   | manager-web            | manager-api            | HTTP REST          | Bearer user_token    | 所有管理操作              |
| 8   | manager-mobile         | manager-api            | HTTP REST          | Bearer user_token    | 移动端管理操作             |
| 9   | Python(xiaozhi-server) | 各AI厂商                  | HTTP(S)/WebSocket  | 各厂商API Key           | ASR/LLM/TTS推理       |
| 10  | manager-api            | MCP端点                  | WebSocket JSON-RPC | URL内加密token          | 工具发现                |

## 4. 认证机制汇总

### 4.1 三种认证场景

```
┌─────────────────────────────────────────────────────────┐
│                    认证场景                              │
├──────────────┬──────────────────┬───────────────────────┤
│ 用户认证     │ 服务间认证        │ 设备认证              │
│ (oauth2)     │ (server secret)  │ (HMAC JWT)            │
├──────────────┼──────────────────┼───────────────────────┤
│ Bearer:      │ Bearer:          │ Bearer:               │
│ SysUserToken │ server.secret    │ HMAC-SHA256签名.时间戳 │
│ (UUID)       │ (系统参数)        │                       │
├──────────────┼──────────────────┼───────────────────────┤
│ 适用:        │ 适用:            │ 适用:                  │
│ 前端→Java    │ Python→Java      │ 设备→Python WebSocket  │
│              │ Java→Python      │ Vision API             │
├──────────────┼──────────────────┼───────────────────────┤
│ 校验方:      │ 校验方:          │ 校验方:                │
│ Oauth2Realm  │ ServerSecret     │ AuthManager            │
│ (DB查Token)  │ Filter           │ (HMAC验签+时间窗)      │
│              │ (字符串比对)      │                        │
└──────────────┴──────────────────┴───────────────────────┘
```

### 4.2 密钥统一

```
app.py 启动时确定 auth_key:
  server.auth_key → manager-api.secret → 随机UUID

该 auth_key 统一用于:
├── WebSocket 设备连接 HMAC 签名
├── OTA 返回的 WebSocket token
├── Vision API 的 JWT 签名
└── 与 manager-api 的 server.secret 保持一致
```

## 5. 数据流向总结

### 5.1 配置数据流

```
管理员(Web/Mobile) → manager-api(MySQL) → Python(运行时内存)
                                        ↑
                                   系统参数 + 智能体 + 模型 聚合
```

### 5.2 业务数据流

```
设备音频 → Python(ASR→文本) → Python(LLM→回复) → Python(TTS→音频) → 设备播放
                                    ↓                     ↓
                              manager-api(MySQL)     manager-api(MySQL)
                              (聊天记录存储)          (聊天记录存储)
                                    ↓
                              前端(聊天记录展示)
```

### 5.3 固件更新数据流

```
管理员上传固件 → manager-api(文件+数据库)
                         ↓
设备OTA请求 → manager-api(版本比对) → 下载URL → 设备下载更新
```

### 5.4 知识库数据流

```
管理员操作 → manager-api → RAGFlow(创建/文档/检索)
                ↓
            本地影子表(MySQL)
```
