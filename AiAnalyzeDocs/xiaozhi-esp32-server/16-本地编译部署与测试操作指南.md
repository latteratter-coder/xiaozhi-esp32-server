# 本地编译部署与测试操作指南

## 一、xiaozhi-deploy 与 deploy 目录的关系

- `/home/one/xiaozhi-deploy/` 是从项目仓库 `deploy/xiaozhi-deploy/` 复制出的独立部署目录，但其中 `docker-compose.yml` 为空文件、`data/.config.yaml` 为0字节
- 项目仓库中的 `deploy/xiaozhi-deploy/` 是完整部署模板，`docker-compose.yml` 使用本地镜像 `xiaozhi-server:local` / `xiaozhi-web:local`
- **`/home/one/xiaozhi-deploy/` 可以删除**，实际数据在 `deploy/xiaozhi-deploy/` 中

## 二、Docker 容器管理

### 2.1 常用命令

```bash
# 查看运行中的容器
sudo docker ps --filter "name=xiaozhi"

# 停止所有容器
sudo docker stop xiaozhi-esp32-server xiaozhi-esp32-server-web xiaozhi-esp32-server-db xiaozhi-esp32-server-redis

# 重启 server 容器（修改配置后）
sudo docker restart xiaozhi-esp32-server

# 删除容器（可选）
sudo docker rm xiaozhi-esp32-server xiaozhi-esp32-server-web xiaozhi-esp32-server-db xiaozhi-esp32-server-redis
```

### 2.2 容器列表

| 容器名 | 镜像 | 端口 | 说明 |
|--------|------|------|------|
| xiaozhi-esp32-server | xiaozhi-server:local | 8000, 8003 | Python 核心服务 |
| xiaozhi-esp32-server-web | xiaozhi-web:local | 8002 | Java API + Vue 前端 |
| xiaozhi-esp32-server-db | mysql:latest | 3306 | MySQL 数据库 |
| xiaozhi-esp32-server-redis | redis:8.0 | 6379 | Redis 缓存 |

## 三、本地编译与 Docker 镜像打包

### 3.1 构建 Server 镜像（Python 后端）

两层 Dockerfile：
- `Dockerfile-server-base`：基础镜像（系统依赖 + Python 包，构建慢）
- `Dockerfile-server`：生产镜像（仅应用代码，基于 base，构建快）

```bash
cd /home/one/xiaozhi-esp32-server
# 第一步：构建基础镜像
sudo docker build -f Dockerfile-server-base -t xiaozhi-server:base .
# 第二步：构建生产镜像
sudo docker build -f Dockerfile-server -t xiaozhi-server:local .
```

### 3.2 构建 Web 镜像（Vue前端 + Java后端）

推荐使用 `Dockerfile-web-local`（Ubuntu基础，兼容性更好）：

```bash
sudo docker build -f Dockerfile-web-local -t xiaozhi-web:local .
```

### 3.3 导出/导入镜像

```bash
# 导出
sudo docker save xiaozhi-server:local -o xiaozhi-server-local.tar
sudo docker save xiaozhi-web:local -o xiaozhi-web-local.tar

# 导入
sudo docker load -i xiaozhi-server-local.tar
sudo docker load -i xiaozhi-web-local.tar
```

## 七、Windows 端口转发

```powershell
# PowerShell (管理员) - 替换 <WSL_IP> 为实际的 WSL IP 地址
netsh interface portproxy add v4tov4 listenport=8000 listenaddress=127.0.0.1 connectport=8000 connectaddress=<WSL_IP>
netsh interface portproxy add v4tov4 listenport=8002 listenaddress=127.0.0.1 connectport=8002 connectaddress=<WSL_IP>
netsh interface portproxy add v4tov4 listenport=8003 listenaddress=127.0.0.1 connectport=8003 connectaddress=<WSL_IP>

netsh interface portproxy show all    # 验证
netsh interface portproxy reset       # 清除

# 获取 WSL IP 命令：在 WSL 中执行 hostname -I | awk '{print $1}'
```

| 端口 | 服务 |
|------|------|
| 8000 | WebSocket |
| 8002 | 智控台 |
| 8003 | HTTP/OTA |

## 十一、常见问题排查

- **服务无法启动**：`sudo docker logs xiaozhi-esp32-server --tail 100` 查看日志
- **WebSocket连接失败**：检查 `ss -tlnp | grep 8000` 和设备白名单配置
- **LLM调用失败**：检查配置中的 API Key 是否有效
- **天气查询失败**：`curl` 测试天气 API 是否可达
